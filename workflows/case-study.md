# Workflow: Case Study Creation

Создание кейса для портфолио в формате **Problem → Solution → Outcome**, а не «мы сделали сайт с N страницами».

> **Принцип:** Кейс продаёт не ваши навыки, а результат для бизнеса клиента.

## Формат кейса

### 1. Headline (1 строка)

Формула: **[Результат] для [клиент/ниша] через [решение]**

Примеры:
- ✅ «Рост конверсии на 34% для e-commerce бизнеса через редизайн воронки»
- ✅ «Автоматизация поддержки: -60% нагрузки на менеджеров через Telegram-бота»
- ❌ «Разработка сайта для компании X»
- ❌ «Создание мобильного приложения»

### 2. About Client (2–3 предложения)

- Что за бизнес, в какой нише
- Масштаб (без NDA-деталей)
- Почему обратились

### 3. Challenge / Problem

- Какая бизнес-проблема существовала
- Сколько эта проблема стоила клиенту (если можно раскрыть)
- Почему предыдущие попытки не сработали (если были)

### 4. Solution (Outcome-focused)

- Какой подход мы выбрали и **почему**
- Ключевые решения, которые повлияли на результат
- Технологии — только если они критичны для понимания outcome

**Формат:** «Мы сделали X, потому что это позволяет клиенту Y»

НЕ: «Мы использовали React, TypeScript, PostgreSQL»

### 5. Outcome / Results

Конкретные метрики:
- До/после сравнение
- Количественные результаты (%, $, время)
- Цитата клиента (если есть)

```markdown
| Метрика                       | До     | После        |
| ----------------------------- | ------ | ------------ |
| Конверсия                     | 2.1%   | 3.8% (+81%)  |
| Время обработки заявки        | 45 мин | 8 мин (-82%) |
| Стоимость привлечения клиента | $120   | $67 (-44%)   |
```

### 6. Client Testimonial (опционально)

Прямая цитата клиента о результате, не о процессе:
- ✅ «Благодаря новой системе мы увеличили оборот на 40% за квартал»
- ❌ «Отличная команда, работали быстро»

## Шаблон кейса

```markdown
# [Headline: Результат для клиента]

## О клиенте
[2-3 предложения о бизнесе]

## Проблема
[Описание бизнес-проблемы и её стоимости]

## Решение
[Что мы сделали и почему — фокус на бизнес-логике, не на технологиях]

## Результат

| Метрика | До  | После |
| ------- | --- | ----- |
| ...     | ... | ...   |

> «[Цитата клиента]» — [Имя, должность]

## Технологии
[Краткий список — внизу, не в фокусе]
```

## Где использовать кейсы

1. **Портфолио на сайте** — основной раздел
2. **Коммерческие предложения** — раздел «Почему мы» (см. `/proposal`)
3. **Социальные сети** — адаптированные посты
4. **Discovery-встречи** — для демонстрации экспертизы

## Чек-лист кейса

- [ ] Headline содержит конкретный результат
- [ ] Есть измеримые метрики до/после
- [ ] Технологии упомянуты минимально и в контексте ценности
- [ ] Клиент узнает свою историю и будет гордиться результатом
- [ ] Кейс понятен нетехническому читателю (CEO/фаундер)
